## 当前状态

网站部署在 GitHub Pages，7 张图片总计约 720KB。已做过一轮基础优化（压缩+懒加载），但加载仍然偏慢。

## 技术约束

- 部署在 GitHub Pages，不支持服务端处理
- 不能使用需要 API Key 的第三方图片 CDN
- 单文件 HTML（CSS/JS 内联），可以加 build 脚本
- 中国大陆用户访问（Google Fonts 等被墙）
- 没有 npm/build 工具链，但 Python 可用
- 图片文件名不能改（HTML 里引用太多处）

## 请你思考并回答

1. 这 7 张图中，哪些应该转 WebP？哪些应该转 AVIF？Github Pages 对这两种格式的兼容性如何？
2. 对于 4 张肥嘟嘟装饰图（PNG 透明背景），有没有比 PNG 更小的透明格式？
3. 是否可以用 CSS/SVG 替代某些装饰图？比如线稿那张？
4. 能否用占位符（blur-up、骨架屏、低质量预览）让页面先出来、图片后加载？
5. 首屏 5 张图（头像+4张肥嘟嘟），能否用 preload、resource hint、或调整加载优先级来加速？
6. 手作照片 277 有 168KB 仍然偏大——在不明显损失画质的前提下，压缩到多少是极限？
7. 以上所有手段叠加，理论上的极限首次加载体积能降到多少？
